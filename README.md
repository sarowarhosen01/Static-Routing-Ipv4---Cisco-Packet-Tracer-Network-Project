# Static Route v4 Alternative - Cisco Packet Tracer Project

## 📋 Project Overview

A complete **static routing** implementation in an IPv4 network using Cisco Packet Tracer. This project serves as an alternative design to standard static routing labs, featuring a robust multi-router topology with proper subnetting and manual route configuration.

## 🛠 Technologies & Tools

- **Cisco Packet Tracer** (Latest version recommended)
- IPv4 Addressing & VLSM Subnetting
- Static Routing (including default routes)
- Cisco IOS Command Line Configuration
- Network Simulation

## 📁 Project Files

- `Static Route v4 altanative.pkt` - Main Packet Tracer project file

## 🏗 Network Topology

- Multiple Routers connected via serial and/or Ethernet links
- Layer 2 Switches for LAN segments
- PC/Server end devices in different subnets
- Full inter-network communication via static routes

## ✨ Features Implemented

- Custom IPv4 addressing scheme
- Static route configuration on all routers
- Default routing for internet/outside connectivity
- Complete end-to-end connectivity testing
- Alternative network design for better understanding of routing concepts

## 🔧 How to Use

1. Open `Static Route v4 altanative.pkt` in Cisco Packet Tracer
2. Check router configurations:
   - `show running-config`
   - `show ip route`
3. Test connectivity:
   - Ping between PCs in different subnets
   - Use `traceroute` to verify path
4. Explore the alternative topology design
